Проект состоит из бэкенда на Node.js/TypeScript и фронтенда на React, которые общаются через REST API.​

Описание проекта
Проект реализует учет мебельной продукции, материалов, цехов и маршрутов изготовления. Бэкенд предоставляет REST API для работы с данными (изделия, материалы, цеха, связи изделие–цех, расчеты времени и сырья), фронтенд на React отображает таблицы и формы для CRUD‑операций и использует эти API для интеграции.​

Технологии
Бэкенд: Node.js, TypeScript, Express, InversifyJS (DI), Prisma ORM, SQLite, Swagger/OpenAPI.​

Фронтенд: React, TypeScript/JavaScript, fetch/axios для работы с REST API.​

Структура репозитория
backend/ – серверная часть (Furniture Accounting API, Prisma‑схема, миграции, seed‑данные, Swagger).​

frontend/ – клиентская часть на React (страницы с таблицами изделий, материалов, цехов, формами создания/редактирования и интеграцией с бэком).

Запуск бэкенда
Перейти в директорию backend и установить зависимости.

Настроить .env с DATABASE_URL для SQLite.​

Выполнить миграции и seed‑скрипт Prisma.​

Запустить dev‑сервер (npm run dev) и убедиться, что API доступно по http://localhost:3000/api, а документация – по http://localhost:3000/api-docs.​

Запуск фронтенда
Перейти в директорию frontend, установить зависимости React‑приложения.​

В конфигурации фронта указать базовый URL бэкенда (например, http://localhost:3000/api).​

Запустить dev‑сервер React (npm start / npm run dev) и открыть приложение в браузере.​

Интеграция фронта и бэка
Фронтенд обращается к REST‑эндпоинтам бэкенда (/products, /materials, /workshops, /product-workshops, /products/:id/production-time, /products/:id/material-consumption) для загрузки данных и отправки изменений в формате JSON.​

Запросы выполняются через fetch/axios, в ответ фронт получает данные в структурированном виде (с вложенными объектами материал/тип/цехи), что позволяет отображать полные таблицы и реализовать сценарии добавления, редактирования и удаления записей.​
